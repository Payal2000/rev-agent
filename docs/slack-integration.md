**Date:** March 2026

## Overview

RevAgent's Slack integration provides two-way communication between your Slack workspace and the multi-agent revenue intelligence pipeline:

- **Outbound** — Anomaly alerts and daily briefings are pushed to Slack channels automatically
- **Inbound** — The `/revagent` slash command lets users query the agent directly from Slack
- **Interactive** — Approve or reject Action Agent recommendations via buttons in Slack messages

---

## Architecture

```
Slack Workspace
    │
    ├── /revagent <query>  ──→  POST /api/slack/events
    │                               │
    │                           Signature verification (HMAC-SHA256)
    │                               │
    │                           LangGraph pipeline (same as web UI)
    │                               │
    │                           Formatted Block Kit response
    │                               │
    │                     ←──  response_url POST (delayed response)
    │
    └── Approve/Reject button  ──→  POST /api/slack/interactions
                                        │
                                    LangGraph Command(resume=...)
                                        │
                                    Graph resumes from interrupt
```

## Outbound Notifications

### Anomaly Alert

Called by the Insights Agent or Celery webhook task when a metric z-score exceeds threshold.

```python
from tools.slack_tools import send_anomaly_alert

await send_anomaly_alert(
    metric_name="MRR",
    explanation="MRR dropped 18% in 24 hours — 3.2σ below 90-day mean.",
    severity="high",         # low | medium | high | critical
    z_score=3.2,
    channel_id="#revenue-ops"  # optional, falls back to SLACK_CHANNEL_ID
)
```

The message uses Slack Block Kit with severity emoji, metric fields, and a "View in RevAgent" CTA button.

### Daily Briefing

Called by the Celery beat scheduler at 6AM via `tasks/scheduled.py`.

```python
from tools.slack_tools import send_daily_briefing

await send_daily_briefing(briefing_text)
```

The briefing is generated by running the full LangGraph pipeline with a predefined prompt, then posted to the configured channel.

## Inbound — Slash Command

### Setup

1. Create a Slack App at [api.slack.com/apps](https://api.slack.com/apps)
2. Add a **Slash Command**: `/revagent` → `https://your-domain.com/api/slack/events`
3. Add **Interactivity** request URL: `https://your-domain.com/api/slack/interactions`
4. Install the app to your workspace and copy the tokens

### Environment Variables

```bash
SLACK_BOT_TOKEN=xoxb-...         # Bot OAuth token
SLACK_SIGNING_SECRET=...          # From App credentials (for signature verification)
SLACK_CHANNEL_ID=#revenue-alerts  # Default outbound channel
```

### Usage

```
/revagent What is our MRR this month?
/revagent Show me churn anomalies in the last 7 days
/revagent Forecast revenue for the next 90 days
/revagent What are the top recommendations to reduce churn?
```

RevAgent responds with:
- A `"Results coming up..."` acknowledgment (within 3 seconds, as required by Slack)
- A follow-up Block Kit message with the full analysis (data, anomalies, forecast, recommendations)

### Response Flow

```
User types /revagent <query>
    ↓
POST /api/slack/events  (Slack sends form-encoded body)
    ↓
HMAC-SHA256 signature verified using SLACK_SIGNING_SECRET
    ↓
Immediate ACK response: "🔍 Analyzing: _{query}_"  (within 3s)
    ↓
asyncio.create_task() → runs LangGraph in background
    ↓
Full agent pipeline runs (Supervisor → Query/Insights/Forecast/Action → Validator)
    ↓
Result formatted as Block Kit blocks
    ↓
POST to response_url (Slack delayed response, up to 30 min)
```

## Interactive Approvals

When the Action Agent pauses for human approval, the Slack message shows **Approve** and **Reject** buttons alongside the recommendation.

```
⏸ This action requires your approval:
  [✅ Approve]  [❌ Reject]
```

Clicking either button sends a payload to `POST /api/slack/interactions`.

### Approval Flow

```
User clicks ✅ Approve (or ❌ Reject)
    ↓
POST /api/slack/interactions  (Slack sends JSON payload)
    ↓
Signature verified
    ↓
action_id = "approve_recommendation" or "reject_recommendation"
value = session_id  (encoded in button)
    ↓
graph.astream(Command(resume={approved: true/false}), config)
    ↓
LangGraph resumes from PostgreSQL checkpoint
    ↓
Response posted back: "✅ Action approved and executed."
```

## Security

- All requests from Slack are verified using **HMAC-SHA256** with `SLACK_SIGNING_SECRET`
- Requests older than **5 minutes** are rejected (replay attack prevention)
- The bot token is never exposed to clients — all API calls happen server-side

## Multi-Tenant Mapping

Slack workspaces are mapped to `company_id` via `_get_company_id_for_team(team_id)` in `slack.py`. Currently falls back to the demo tenant. In production, add a `slack_workspaces` table:

```sql
CREATE TABLE slack_workspaces (
    team_id TEXT PRIMARY KEY,
    company_id UUID REFERENCES companies(id),
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```
